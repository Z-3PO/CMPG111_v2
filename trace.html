Trace & Debug

Two essential skills: predicting what code will output, and finding bugs in broken code. Work through these on paper first!

🧠 Part A: What Does This Code Output?

Read each snippet carefully, trace the variables, and predict the exact output before revealing the answer.

Easy Input/Output

T1 · Variable Swap

What is the output?

x = 10
y = 20
x = y
y = x
print(x, y)
✅ Output
20 20
After x = y, both are 20. Then y = x assigns 20 to y again. Common trap: students think this swaps the values, but the original x (10) is lost. A real swap needs a temp variable: temp = x; x = y; y = temp.

x = y2020
y = x2020
Easy Input/Output

T2 · String Arithmetic

What is the output?

a = "3"
b = "7"
print(a + b)
print(int(a) + int(b))
✅ Output
37 10
"3" + "7" concatenates strings → "37". But int("3") + int("7") adds numbers → 10. This is the #1 input mistake: input() returns strings, so + concatenates instead of adding unless you convert.
Medium Decisions

T3 · Nested If Tracing

If the user enters 25, what is the output?

age = int(input("Age: "))
if age >= 18:
    if age >= 65:
        print("Senior")
    else:
        print("Adult")
elif age >= 13:
    print("Teen")
else:
    print("Child")
print("Done")
✅ Output
Adult Done
25 ≥ 18 → enters outer if. 25 is NOT ≥ 65 → inner else prints "Adult". The elif and else are skipped entirely (they belong to the outer if). "Done" prints because it's outside all blocks (no indentation).
Medium Decisions

T4 · Boolean Logic

What is the output?

x = 5
y = 10
print(x > 3 and y < 8)
print(x > 3 or y < 8)
print(not (x == 5))
✅ Output
False True False
Line 1: True and False → False (both must be True for and).
Line 2: True or False → True (one is enough for or).
Line 3: not True → False.
Medium Loops

T5 · For Loop Accumulator

What is the output?

total = 0
for i in range(1, 4):
    total = total + i * 2
print(total)
✅ Output
12
ii * 2total
120 + 2 = 2
242 + 4 = 6
366 + 6 = 12
range(1, 4) gives 1, 2, 3. Each iteration multiplies i by 2 and adds to total. Common mistake: including 4 in the range — remember, stop value is excluded.
Hard Loops

T6 · While Loop Counter

What is the output?

n = 1
while n < 20:
    n = n * 2
print(n)
✅ Output
32

n doubles each iteration: 1→2→4→8→16→32. When n=32, the condition 32 < 20 is False, so the loop stops. The print(n) is OUTSIDE the loop, so it prints 32 (not 16!).
Hard Functions

T7 · Scope & Return

What is the output?

def mystery(x):
    x = x + 10
    return x * 2

x = 5
result = mystery(x)
print(x)
print(result)
✅ Output
5 30
The x inside the function is local — a separate copy. Changing it doesn't affect the x = 5 in the main program. Inside the function: 5 + 10 = 15, then 15 × 2 = 30 returned. The global x stays 5.
Medium Functions

T8 · print() vs return

What is the output?

def double(n):
    print(n * 2)

result = double(7)
print(result)
✅ Output
14 None
double(7) prints 14 inside the function. But since there's no return statement, the function returns None by default. So result = None, and the second print shows None. This is why return matters!
Medium Files/Exceptions

T9 · try/except Flow

If the user enters "abc", what is the output?

try:
    num = int(input("Enter: "))
    print("Got", num)
except ValueError:
    print("Bad input")
else:
    print("Success")
finally:
    print("Done")
✅ Output
Bad input Done
int("abc") raises ValueError → jumps to except → prints "Bad input". The else only runs when there's NO error, so it's skipped. finally ALWAYS runs, so "Done" prints. The line print("Got", num) was never reached because the exception happened on the same line.
Medium Lists/Tuples

T10 · List Operations

What is the output?

nums = [10, 20, 30, 40, 50]
nums.append(60)
nums.remove(30)
nums.pop(0)
print(nums)
print(nums[1:3])
✅ Output
[20, 40, 50, 60] [40, 50]
OperationList
Start[10, 20, 30, 40, 50]
append(60)[10, 20, 30, 40, 50, 60]
remove(30)[10, 20, 40, 50, 60]
pop(0)[20, 40, 50, 60]
pop(0) removes and returns the item at index 0 (which is 10). Then nums[1:3] slices indexes 1 and 2 (not 3!) → [40, 50].
Hard Lists/Tuples

T11 · Tuple Unpacking in Loop

What is the output?

students = [("Ann", 85), ("Bob", 72), ("Cal", 91)]
total = 0
for name, mark in students:
    total += mark
print(f"Average: {total / len(students)}")
✅ Output
Average: 82.66666666666667
Each tuple in the list is unpacked into name and mark. Total = 85 + 72 + 91 = 248. Average = 248 / 3 ≈ 82.67. Note: without :.2f formatting, Python shows all decimal places.
Hard Database

T12 · SQL Query Result

Given a table items with rows (1,"Pen",5.50), (2,"Book",89.99), (3,"Ruler",12.00), (4,"Eraser",8.50) — what does this print?

cur.execute("SELECT name, price FROM items WHERE price > 10 ORDER BY price DESC")
for row in cur.fetchall():
    print(row)
✅ Output
('Book', 89.99) ('Ruler', 12.0)
WHERE price > 10 excludes Pen (5.50) and Eraser (8.50). ORDER BY price DESC sorts highest first. fetchall() returns tuples, so each row prints as ('name', price).

🐛 Part B: Find & Fix the Bug

Each snippet has exactly ONE bug. Find it, explain why it's wrong, and write the fix.

Easy Input/Output

B1 · The Total Won't Add Up

The programmer wants to add two numbers, but the output is wrong. Fix it.

a = input("First number: ")
b = input("Second number: ")
total = a + b
print(f"Total: {total}")   # Typing 3 and 5 gives "35"!
🐛 The Bug

input() returns strings. "3" + "5" concatenates to "35" instead of adding to 8.

❌ Wrong 
a = input("First number: ")
b = input("Second number: ")
✅ Fixed 
a = int(input("First number: "))
b = int(input("Second number: "))
Easy Decisions

B2 · Assignment vs Comparison

This crashes with a SyntaxError. Why?

password = input("Password: ")
if password = "secret123":
    print("Access granted")
🐛 The Bug

Uses = (assignment) instead of == (comparison).

❌ Wrong 
if password = "secret123":
✅ Fixed 
if password == "secret123":
Medium Decisions

B3 · Wrong Grade Boundaries

A student with 90% is getting "Pass" instead of "Distinction". Fix the logic.

mark = int(input("Mark: "))
if mark >= 50:
    print("Pass")
elif mark >= 75:
    print("Distinction")
else:
    print("Fail")
🐛 The Bug

90 ≥ 50 is True, so the first branch runs and elif is never checked. You must check the highest condition first.

✅ Fixed 
if mark >= 75:
    print("Distinction")
elif mark >= 50:
    print("Pass")
else:
    print("Fail")
Easy Loops

B4 · Infinite Loop

This program never stops. Find the missing line.

count = 1
while count <= 5:
    print(f"Count: {count}")
🐛 The Bug

count never changes, so count <= 5 stays True forever.

✅ Fixed — add counter update inside the loop 
count = 1
while count <= 5:
    print(f"Count: {count}")
    count += 1    # THIS WAS MISSING!
Medium Loops

B5 · Off-by-One Error

The program should print 1 to 10, but stops at 9.

for i in range(1, 10):
    print(i)
🐛 The Bug

range(1, 10) stops BEFORE 10 (gives 1–9). To include 10, use range(1, 11).

✅ Fixed 
for i in range(1, 11):
    print(i)
Medium Functions

B6 · Can't Use the Result

The programmer expects result to hold a number, but gets None.

def calculate_area(length, width):
    area = length * width
    print(area)

result = calculate_area(5, 3)
print(f"Double: {result * 2}")  # TypeError!
🐛 The Bug

The function uses print() instead of return. print() displays but returns None. You can't multiply None * 2.

✅ Fixed 
def calculate_area(length, width):
    area = length * width
    return area
Medium Files/Exceptions

B7 · Writing After Close

Only the first line appears in the file. Why?

with open("log.txt", "w") as f:
    f.write("Line 1\n")

f.write("Line 2\n")  # ValueError: I/O on closed file
🐛 The Bug

The with block ends after the first write (notice the indentation). By line 4, the file is already closed. Both writes must be inside the with block.

✅ Fixed 
with open("log.txt", "w") as f:
    f.write("Line 1\n")
    f.write("Line 2\n")
Medium Lists/Tuples

B8 · IndexError Mystery

This crashes on the last iteration. Find why.

names = ["Ann", "Bob", "Cal"]
for i in range(1, len(names) + 1):
    print(names[i])
🐛 The Bug

range(1, 4) gives i = 1, 2, 3. But names[3] doesn't exist — a 3-item list has indexes 0, 1, 2. Also, index 0 ("Ann") is skipped entirely.

✅ Fixed — start at 0 
for i in range(len(names)):
    print(names[i])
Or even simpler: for name in names: print(name)
Medium Database

B9 · Data Disappears

The programmer inserts data, but after re-running the program, the table is empty. Why?

import sqlite3
conn = sqlite3.connect("shop.db")
cur = conn.cursor()
cur.execute("INSERT INTO products(name,price) VALUES (?,?)", ("Pen", 10))
conn.close()
🐛 The Bug

Missing conn.commit()! Without it, the INSERT is never saved to disk.

✅ Fixed — add commit before close 
cur.execute("INSERT INTO products(name,price) VALUES (?,?)", ("Pen", 10))
conn.commit()   # THIS WAS MISSING!
conn.close()
Hard Database

B10 · SQL Injection Risk

This code works, but has a critical security vulnerability. Fix it.

name = input("Name: ")
cur.execute(f"SELECT * FROM users WHERE name = '{name}'")
🐛 The Bug

Using f-strings to build SQL allows SQL injection. A user could type ' OR 1=1 -- to see all records. Always use ? placeholders.

✅ Fixed 
name = input("Name: ")
cur.execute("SELECT * FROM users WHERE name = ?", (name,))
