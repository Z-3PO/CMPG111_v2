📥 Input, Processing & Output

The three-step recipe every program follows

Core Concept

The IPO Model

🍳 A program is like a chef: Input = ingredients, Processing = cooking, Output = the dish.
INPUT → PROCESS → OUTPUT
Syntax

Variables & Data Types

name = "Alice"        # str
age  = 21             # int
gpa  = 3.85           # float
enrolled = True      # bool
📝 Tip: Variable names are case-sensitive. Age ≠ age!
Syntax

Getting Input

# input() always returns a string
name = input("Name? ")

# Convert for numbers:
age   = int(input("Age: "))
price = float(input("Price: "))
⚠️ #1 Mistake: Forgetting to convert! You can't do maths on strings.
Syntax

Displaying Output

print("Hello, World!")
print("Name:", name, "Age:", age)

# f-strings ⭐
print(f"Hello {name}, you are {age}.")

# Format decimals
print(f"Total: R{price:.2f}")
Reference

Arithmetic Operators

+Addition5+3=8
-Subtraction10-4=6
*Multiplication3*7=21
/Division7/2=3.5
//Floor div7//2=3
%Modulus7%2=1
**Exponent2**3=8
📝 Tip: num % 2 == 0 means even!

🔀 Decision Structures & Boolean Logic

Teaching your program to make choices

Core Concept

What Is a Decision?

🚦 A decision is like a traffic light: check a condition, take different paths based on True/False.
Reference

Comparison Operators

🔄 Repetition Structures

Making the computer do the boring stuff over and over

Core Concept

Why Loops?

🏋️ Doing 100 push-ups: don't write 100 instructions. Say "do a push-up" and repeat 100 times.
  • while — repeats while condition is True
  • for — repeats a fixed number of times
Syntax

while Loop

count = 1
while count <= 5:
    print(f"Count: {count}")
    count += 1  # essential!
⚠️ Infinite Loop Alert: Forget to update the counter and it runs forever!
Syntax

for Loop & range()

for i in range(5):
    print(i)        # 0,1,2,3,4

for i in range(1, 6):
    print(i)        # 1,2,3,4,5

for i in range(0, 10, 2):
    print(i)        # 0,2,4,6,8
📝 Tip: range() never includes the stop value!
Pattern

Accumulator & Counter

# Running total
total = 0
for i in range(5):
    num = int(input("Number: "))
    total += num

# Counting matches
passes = 0
for i in range(10):
    mark = int(input("Mark: "))
    if mark >= 50:
        passes += 1
Syntax

Sentinel Loop

total = 0
num = int(input("Num (-1 to quit): "))
while num != -1:
    total += num
    num = int(input("Num (-1 to quit): "))
print(f"Total: {total}")
🔑 Pattern: input() appears TWICE (priming read + inside loop).

🧩 Functions

Write once, use anywhere — your reusable code blocks

Core Concept

What Is a Function?

🧃 A juice machine: fruit in (arguments), squeeze (processing), juice out (return value). Same machine, used many times.
Syntax

Define & Call

# Define (the recipe)
def greet(name):
    print(f"Hello, {name}!")

# Call (use the recipe)
greet("Alice")
greet("Bob")
def = DEFINE → return = OUTPUT
Syntax

Parameters vs Arguments

def add(a, b):    # a,b = PARAMETERS
    return a + b

result = add(3, 7)  # 3,7 = ARGUMENTS
print(result)       # 10
📝 Memory Trick: Parameter = Placeholder, Argument = Actual value.
Logic

return vs print

# ❌ Shows, doesn't give back
def area_bad(r):
    print(3.14 * r ** 2)

# ✅ Gives value for reuse
def area(r):
    return 3.14 * r ** 2

a = area(5)       # store
d = area(5) * 2   # use in math
⚠️ Key: Question says "calculate/return" → use return. Says "display" → use print().
Logic

Variable Scope

def my_fn():
    secret = 42    # LOCAL
    print(secret)   # ✅ works

my_fn()
print(secret)       # ❌ ERROR!
🏠 Local = items inside a house. They don't exist on the street (main program).

📁 Files & Exceptions

Reading/writing files and handling errors gracefully

Core Concept

Why Files?

💾 Variables vanish when the program ends. Files are like a filing cabinet — your data stays even after you close the program.

Files let you save data permanently: scores, settings, records, logs — anything that should survive between runs.

Syntax

Writing to a File

# Open, write, close
file = open("notes.txt", "w")
file.write("Hello!\n")
file.write("Line 2\n")
file.close()

# Better — with statement (auto-closes)
with open("notes.txt", "w") as f:
    f.write("Hello!\n")
Syntax

Reading from a File

# Read entire file
with open("notes.txt", "r") as f:
    content = f.read()
print(content)

# Read line by line
with open("notes.txt", "r") as f:
    for line in f:
        print(line.strip())
Reference

File Modes

ModeMeaningIf file exists
"r"ReadOpens it
"w"Write⚠️ Erases contents!
"a"AppendAdds to end
"r+"Read+WriteOpens it
⚠️ Warning: "w" mode wipes the file! Use "a" to add to an existing file.
Core Concept

What Is an Exception?

💥 An exception is like a tripwire. When something goes wrong (bad input, missing file, divide by zero), Python raises an exception. Without a "safety net", your program crashes.
Syntax

try / except

try:
    age = int(input("Age: "))
    print(f"You are {age}")
except ValueError:
    print("Please enter a number!")

# Multiple exceptions
try:
    num = int(input("Num: "))
    result = 100 / num
except ValueError:
    print("Not a number")
except ZeroDivisionError:
    print("Cannot divide by zero")
Reference

Common Exceptions

ExceptionTriggered by

ZeroDivisionError5 / 0
FileNotFoundErrorOpening missing file

Syntax

try / except / else / finally

try:
    f = open("data.txt", "r")
    data = f.read()
except FileNotFoundError:
    print("File missing")
else:
    print("Read OK")     # if no error
finally:
    print("Done")        # always runs

📋 Lists & Tuples

Storing collections of data

Core Concept

What Is a List?

🚃 A list is like a train — each carriage has a number (index) and holds something. You can add, remove, and rearrange carriages anytime.
fruits = ["apple", "banana", "cherry"]
scores = [85, 92, 78, 90]
mixed  = ["Alice", 21, True]
empty  = []
Syntax

Indexing & Slicing

fruits = ["apple", "banana", "cherry"]

print(fruits[0])    # apple (first)
print(fruits[2])    # cherry
print(fruits[-1])   # cherry (last)

# Slicing [start:stop]
print(fruits[0:2])  # ['apple','banana']
# Note: stop index NOT included
📝 Tip: Indexing starts at 0, not 1. Negative indexes count from the end.
Syntax

Modifying Lists

scores = [85, 92, 78]

scores[0] = 100          # change
scores.append(95)         # add to end
scores.insert(1, 88)      # add at index
scores.remove(78)         # delete value
last = scores.pop()       # remove & return last
Reference

Common List Methods

Syntax

Looping Through Lists

fruits = ["apple", "banana", "cherry"]

# Simple for loop
for fruit in fruits:
    print(fruit)

# With index using enumerate
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# Check membership
if "apple" in fruits:
    print("Found it!")
Core Concept

Tuples — Like Lists, but Locked

# Tuple uses () instead of []
point = (10, 20)
colors = ("red", "green", "blue")

print(point[0])    # 10 (indexing works)

point[0] = 5       # ❌ ERROR! Can't change

# Tuple unpacking
x, y = point      # x=10, y=20
🔑 Lists vs Tuples: Lists = changeable ([]). Tuples = fixed (()). Use tuples for data that shouldn't change: coordinates, RGB colors, dates.
Exam Practice

🎯 Score Analyzer

Ask for 5 scores, store in a list, then display: highest, lowest, average, and total count above 80.

scores = []
for i in range(5):
    s = int(input(f"Score {i+1}: "))
    scores.append(s)

print(f"Highest: {max(scores)}")
print(f"Lowest:  {min(scores)}")
print(f"Average: {sum(scores)/len(scores):.1f}")

above_80 = 0
for s in scores:
    if s > 80:
        above_80 += 1
print(f"Above 80: {above_80}")

🗄️ Database Programming (SQLite)

Storing and querying structured data

Core Concept

What Is a Database?

📚 A database is like a super-organised filing system. Instead of one messy file, data is stored in tables (spreadsheets) with rows and columns. You ask questions (queries) to get back exactly what you need.

SQLite is a lightweight database that comes built into Python — no installation needed!

Syntax

Connecting to a Database

import sqlite3

# Connect (creates file if missing)
conn = sqlite3.connect("school.db")

# Cursor — your helper to run SQL
cur = conn.cursor()

# ... do stuff ...

conn.commit()   # save changes
conn.close()    # disconnect
Syntax

Creating a Table

cur.execute("""
    CREATE TABLE IF NOT EXISTS students (
        id INTEGER PRIMARY KEY,
        name TEXT NOT NULL,
        age INTEGER,
        grade REAL
    )
""")
conn.commit()
📝 Tip: PRIMARY KEY auto-assigns a unique ID. IF NOT EXISTS prevents errors if the table already exists.
Syntax

INSERT — Adding Data

# Using ? placeholders (SAFE way)
cur.execute(
    "INSERT INTO students (name, age, grade) VALUES (?, ?, ?)",
    ("Alice", 20, 85.5)
)
conn.commit()

# Inserting many rows
students = [
    ("Bob", 21, 78.0),
    ("Cara", 19, 92.3)
]
cur.executemany(
    "INSERT INTO students (name, age, grade) VALUES (?, ?, ?)",
    students
)
conn.commit()
Syntax

SELECT — Reading Data

# Get all rows
cur.execute("SELECT * FROM students")
rows = cur.fetchall()
for row in rows:
    print(row)

# Get one row
cur.execute("SELECT * FROM students WHERE id=?", (1,))
one = cur.fetchone()

# Specific columns + condition
cur.execute(
    "SELECT name, grade FROM students WHERE grade > ?",
    (80,)
)
for name, grade in cur.fetchall():
    print(f"{name}: {grade}")
Syntax

UPDATE & DELETE

# UPDATE — change existing rows
cur.execute(
    "UPDATE students SET grade=? WHERE name=?",
    (95.0, "Alice")
)
conn.commit()

# DELETE — remove rows
cur.execute(
    "DELETE FROM students WHERE age < ?",
    (18,)
)
conn.commit()
⚠️ Danger: DELETE or UPDATE without a WHERE clause affects EVERY row!
Reference

Exam Practice

🎯 Student Marks System

Create a database with a students table. Add 3 students. Display all students whose grade is ≥ 80.

import sqlite3

conn = sqlite3.connect("school.db")
cur = conn.cursor()

cur.execute("""CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY, name TEXT, grade REAL)""")

# Insert data
data = [("Ann", 85), ("Bob", 72), ("Cy", 91)]
cur.executemany(
    "INSERT INTO students (name, grade) VALUES (?, ?)", data)
conn.commit()

# Query
cur.execute("SELECT name, grade FROM students WHERE grade >= ?", (80,))
for name, grade in cur.fetchall():
    print(f"{name}: {grade}")

conn.close()
