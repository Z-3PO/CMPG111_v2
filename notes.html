📥 Chapter 1 · Input, Processing & Output

The fundamental building blocks every program is made of

💻 What Is a Computer Program?

+

A computer is an electronic machine that processes data. It can't think for itself — it follows step-by-step instructions called a program. The person who writes these instructions is called a programmer.

Think of a program as a very detailed recipe. A recipe tells a chef exactly what to do, step by step. A program tells a computer exactly what to do, step by step. Python is the language we use to write those instructions.

🔑 Key Terms
Hardware — the physical parts (keyboard, screen, memory).
Software — the programs/instructions that run on the hardware.
Programming language — a special language (like Python) that lets humans write instructions computers can follow.

🔄 The IPO Model

+

Every program, no matter how small or how large, follows three fundamental steps:

  1. Input — Gather data. This usually comes from the user typing via input().
  2. Processing — Do something with the data. Calculate, compare, transform, or rearrange it.
  3. Output — Show the result. Display using print(), or save to a file.
💡 Real-World Analogy: Ordering food at a restaurant. Input = you tell the waiter your order. Processing = the kitchen prepares the food. Output = the waiter brings you the finished meal.

📦 Variables & Data Types

+

A variable is a named storage location in the computer's memory — a labelled box that holds a value. The label is the variable's name; what's inside is its value.

student_name = "Thabo"       # str — text
age          = 19            # int — whole number
average      = 78.5          # float — decimal
is_enrolled  = True          # bool — True/False

Variable Naming Rules
  • Must start with a letter or underscore (not a digit)
  • Can contain letters, digits, and underscores — no spaces or dashes
  • Cannot be a Python keyword (if, for, class, etc.)
  • Case-sensitive: Name, name, NAME are all different
📝 Exam Tip: Use descriptive names. student_age beats x every time. Use snake_case — the Python convention.

🔀 Chapter 2 · Decision Structures & Boolean Logic

Teaching your programs to make choices

✅ Boolean Values

+

In everyday life, many questions have only two possible answers: yes or no, on or off, true or false. In Python, the Boolean data type has exactly two values: True and False (always capitalised).

Every decision in your program ultimately comes down to a Boolean question: "Is this condition True or False?" Based on the answer, the program takes one path or another.

💡 Analogy: A light switch is either ON (True) or OFF (False). No "maybe" or "half-on". Programming decisions work the same way.

⚖️ Comparison Operators

+

⚠️ The #1 Exam Mistake: = is ASSIGNMENT ("put 5 into x"). == is COMPARISON ("is x equal to 5?"). Writing if x = 5: is a syntax error!

🛤️ if / elif / else

+
Simple if — one path
if temperature > 30:
    print("It's a hot day!")
if-else — two paths
if age >= 18:
    print("Can vote")
else:
    print("Too young")
if-elif-else — multiple paths

Python checks each condition top-to-bottom. The first True condition runs — the rest are skipped.

if mark >= 75:
    print("Distinction")
elif mark >= 60:
    print("Merit")
elif mark >= 50:
    print("Pass")
else:
    print("Fail")
📝 Order Matters: Start with the highest/most specific condition first. If you check mark >= 50 before mark >= 75, a student with 80 would be told "Pass" instead of "Distinction"!
🔑 Indentation Is Everything: Python uses indentation (usually 4 spaces) to know which lines belong to the if/else block.

🪆 Nested Decisions

+

A nested decision is an if inside another if. The inner decision only runs if the outer condition is True.

member = input("Are you a member? (yes/no): ")
age = int(input("Enter your age: "))

if member == "yes":
    if age >= 60:
        print("Senior member: 30% off")
    else:
        print("Member: 15% off")
else:
    print("No discount")
📝 Tracing Strategy: Work from OUTSIDE in. Determine if the outer condition is True — only then look at the inner one.

🔄 Chapter 3 · Repetition Structures

Making the computer repeat tasks — so you don't have to

🔁 The while Loop

+

The while loop is condition-controlled. It keeps repeating as long as the condition is True. The moment it becomes False, the loop stops.

💡 Analogy: "While the pot hasn't boiled, keep waiting." Check → not boiling → wait. Check → boiling → stop. 
counter = 1                   # 1. Initialise
while counter <= 5:            # 2. Condition
    print(f"Lap {counter}")     # 3. Body
    counter += 1              # 4. Update (CRITICAL!)
PartPurposeForget and…
InitialiseSet starting valueNameError
ConditionChecked before each iterationLoop won't stop
UpdateMove toward False conditionINFINITE LOOP 💀

🔢 The for Loop and range()

+

The for loop is count-controlled — use it when you know in advance how many times to repeat. No risk of infinite loops!

for i in range(5):          # 0, 1, 2, 3, 4
for i in range(1, 6):       # 1, 2, 3, 4, 5
for i in range(2, 11, 2):   # 2, 4, 6, 8, 10
for i in range(10, 0, -1): # 10, 9, ..., 1
🔑 Key Rule: range() NEVER includes the stop value. Think of it as a fence — you stop BEFORE reaching it.
while vs for — Which to Use?
Use for when…Use while when…
You know how many timesYou don't know — depends on condition
"Repeat 10 times", "for each student""Keep going until user says stop"

🧮 Essential Loop Patterns

+

These patterns appear in almost every exam. Memorise them!

Pattern 1: The Accumulator (Running Total)
total = 0
for i in range(5):
    num = float(input("Enter number: "))
    total += num
print(f"Sum: {total}, Average: {total/5:.2f}")
Pattern 2: The Counter
pass_count = 0
for i in range(10):
    mark = int(input(f"Student {i+1}: "))
    if mark >= 50:
        pass_count += 1
Pattern 3: Finding Max/Min
highest = 0
lowest  = 999999      # start high
for i in range(5):
    score = int(input("Score: "))
    if score > highest: highest = score
    if score < lowest:  lowest  = score

🛑 Sentinel-Controlled Loops

+

A sentinel is a special value that signals the end of input. The loop keeps going until the user types this value.

total = 0
count = 0
num = float(input("Number (-1 to stop): "))
while num != -1:
    total += num
    count += 1
    num = float(input("Number (-1 to stop): "))

if count > 0:
    print(f"Average: {total/count:.2f}")
🔑 Pattern: input() appears twice — once before the loop (priming read) and once inside. This ensures the sentinel is checked before being processed.

🧩 Chapter 4 · Functions

Organising code into reusable, manageable blocks

🤔 Why Functions?

+

Imagine calculating the area of a circle in 5 places. Without functions, you'd copy-paste the formula 5 times. If a bug appears, you'd fix it in 5 places.

💡 Analogy: A function is like a vending machine. Put in money (arguments), press a button (call the function), get a drink (return value). Same machine, used by everyone, every time.
BenefitExplanation
ReusabilityWrite once, use many times
OrganisationBreak big programs into pieces
DebuggingFix in one place = fixed everywhere
ReadabilityWell-named functions explain themselves

🔧 Defining & Calling Functions

+
Defining (writing the recipe)
def greet(name, age):
    print(f"Hello {name}! You are {age}.")
Calling (using the recipe)
greet("Lerato", 22)   # pass values
greet("Sipho", 19)    # reuse with different values
⚠️ Common Mistake: Python reads top-to-bottom. You must DEFINE a function BEFORE you CALL it.

🔒 Variable Scope

+

Variables created inside a function are local — they only exist while the function is running.

def my_function():
    secret = 42    # LOCAL
    print(secret)   # ✅ works inside

my_function()
print(secret)       # ❌ ERROR — doesn't exist here
💡 Analogy: Local variables are like notes on a whiteboard in a meeting room — they vanish when the meeting ends. Global variables are like a hallway notice board — everyone can see.

📁 Chapter 5 · Files & Exceptions

Saving data permanently and handling errors gracefully

💾 Why Do We Need Files?

+

Every variable in your program lives in memory (RAM). The second your program ends, that memory is wiped clean — all data is gone.

Files solve this by storing data on the hard drive, where it stays until you delete it. High scores in a game, student records, configuration settings — anything that needs to survive between program runs lives in a file.

💡 Analogy: Variables are like notes on a whiteboard — wiped clean when you leave the room. Files are like documents in a filing cabinet — still there when you come back tomorrow.

📝 Writing to Files

+

To work with a file, you follow three steps: open it, read/write to it, then close it.

# The manual way
file = open("notes.txt", "w")
file.write("Hello, World!\n")
file.write("Line 2\n")
file.close()

# The better way — 'with' statement auto-closes
with open("notes.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("Line 2\n")
# file is automatically closed here, even if error occurs
File Modes
ModeNameBehaviour
"r"ReadDefault. Error if file missing.
"w"WriteCreates new file OR erases existing content!
"a"AppendAdds to end of file, keeps existing content
"r+"Read+WriteBoth — file must exist
⚠️ Critical Warning: Opening a file with "w" mode ERASES its existing contents immediately. If you want to add to a file, use "a" (append).
📝 Exam Tip: write() does NOT automatically add a newline. You must include \n yourself if you want each piece of data on a new line.

📖 Reading from Files

+ 
# Method 1: Read entire file as one string
with open("notes.txt", "r") as f:
    content = f.read()
print(content)

# Method 2: Read all lines into a list
with open("notes.txt", "r") as f:
    lines = f.readlines()
for line in lines:
    print(line.strip())   # .strip() removes \n

# Method 3: Loop line-by-line (best for big files)
with open("notes.txt", "r") as f:
    for line in f:
        print(line.strip())
📝 Why .strip()? When you read a file, each line keeps its \n newline character. .strip() removes it (plus any spaces at either end) to give you clean text.

💥 What Is an Exception?

+

An exception is Python's way of saying "something went wrong". When Python encounters an unexpected situation — the user types text instead of a number, a file is missing, you try to divide by zero — it raises an exception.

If you don't handle it, the program crashes with an error message. Not great for user experience!

💡 Analogy: Exceptions are like tripwires. Without a safety net (try/except), your program falls straight off the edge when it trips one. With a safety net, you catch it gracefully and keep going.

🛡️ try / except — The Safety Net

+ 
try:
    age = int(input("Your age: "))
    print(f"You are {age}")
except ValueError:
    print("That wasn't a number!")

If the user types "abc", Python would normally crash with ValueError. With try/except, we catch it and show a friendly message instead.

Catching Multiple Exceptions
try:
    num = int(input("Enter a number: "))
    result = 100 / num
    print(f"100 / {num} = {result}")
except ValueError:
    print("Not a valid number")
except ZeroDivisionError:
    print("Cannot divide by zero")
Common Exceptions

🏁 try / except / else / finally — The Full Picture

+ 
try:
    f = open("data.txt", "r")
    data = f.read()
except FileNotFoundError:
    print("File doesn't exist")
else:
    print("Read successfully!")   # only if NO error
finally:
    print("Cleanup done")         # ALWAYS runs

🔑 Use Case: finally is perfect for cleanup code like closing files or database connections — things that MUST happen whether the operation succeeded or not.

📋 Chapter 6 · Lists & Tuples

Collections of data in a single variable

📦 Why Do We Need Lists?

+

Imagine storing the marks of 30 students. You could create 30 variables: mark1, mark2, … mark30. Messy, right? Now imagine 300 students!

A list lets you store many values in one variable. You can add, remove, change, or loop through them easily.

💡 Analogy: A list is like a train. Each carriage has a position number (index) and holds something. You can add carriages, remove them, or swap what's inside. 
fruits = ["apple", "banana", "cherry"]
scores = [85, 92, 78, 90, 88]
mixed  = ["Alice", 21, 85.5, True]
empty  = []     # start empty, add later

🎯 Indexing & Slicing

+

Every item in a list has a position number called its index. Python indexes start at 0, not 1!

fruits = ["apple", "banana", "cherry", "date"]
# Index:      0         1          2         3
# Negative:  -4        -3         -2        -1

print(fruits[0])      # apple  (first)
print(fruits[2])      # cherry
print(fruits[-1])     # date   (last)
print(fruits[-2])     # cherry (second-last)
Slicing — Getting a Range
nums = [10, 20, 30, 40, 50]

print(nums[1:4])    # [20, 30, 40]
print(nums[:3])     # [10, 20, 30]  (from start)
print(nums[2:])     # [30, 40, 50]  (to end)
print(nums[:])      # full copy
📝 Slicing Rule: Just like range(), the stop index is NOT included. nums[1:4] gives indexes 1, 2, 3 — not 4.

🔧 Modifying Lists — Methods

+ 
scores = [85, 92, 78]

scores[0] = 100           # change: [100, 92, 78]
scores.append(95)          # add to end: [100, 92, 78, 95]
scores.insert(1, 88)       # insert at index 1
scores.remove(78)          # remove first 78
last = scores.pop()        # remove & return last
scores.sort()               # sort ascending
scores.sort(reverse=True)   # sort descending
scores.reverse()            # reverse order
Useful Built-in Functions
nums = [15, 23, 8, 42, 16]
print(len(nums))      # 5  (how many)
print(sum(nums))      # 104 (total)
print(max(nums))      # 42  (biggest)
print(min(nums))      # 8   (smallest)

# Check if item is in list
if 23 in nums:
    print("Found!")

🔁 Looping Through Lists

+ 
fruits = ["apple", "banana", "cherry"]

# Method 1: Loop through items directly
for fruit in fruits:
    print(fruit)

# Method 2: Loop using index
for i in range(len(fruits)):
    print(f"{i}: {fruits[i]}")

# Method 3: Both — with enumerate()
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")
📝 Tip: Method 1 is simplest. Use Method 2/3 when you need the index (e.g., for numbering or modifying by position).

🔒 Tuples — Lists That Can't Change

+

A tuple is like a list, but immutable — once created, you can't change its contents. Use round brackets () instead of square brackets [].

point  = (10, 20)              # x, y coordinates
colors = ("red", "green", "blue")
date   = (2024, 6, 15)           # year, month, day

print(point[0])     # 10 — indexing works
print(len(colors))   # 3 — len works

point[0] = 5       # ❌ ERROR! Tuples can't change
point.append(30)  # ❌ ERROR! No append method
Tuple Unpacking — A Superpower
point = (10, 20)
x, y = point              # x=10, y=20

# Common pattern: return multiple values
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([5, 2, 8, 1])  # lo=1, hi=8
Lists vs Tuples — When to Use Which
List []Tuple ()
Changeable?YesNo
SyntaxSquare bracketsRound brackets
MethodsMany (append, pop…)Few
Use forData that will changeData that shouldn't change
🔑 When to use tuples: coordinates (x, y), RGB colors (255, 128, 0), dates, database rows, or any "package" of related values that belong together and shouldn't be modified.

🗄️ Chapter 7 · Database Programming (SQLite)

Storing structured data permanently using SQL

🤔 What Is a Database?

+

A database is an organised collection of data, stored in structured tables that can be queried (asked questions) efficiently.

While files can also store data, databases give you:

  • Built-in structure (tables with specific columns and types)
  • Fast searching, filtering, and sorting of millions of records
  • Data integrity (e.g., age must be a number, IDs must be unique)
  • A standard query language — SQL
💡 Analogy: A file is like a notebook — you write things down in order. A database is like a super-organised filing system with labelled drawers, folders, and an index. Finding anything takes seconds, no matter how big.

🔌 SQLite & sqlite3

+

SQLite is a lightweight, file-based database. Instead of running a separate database server, your entire database lives in a single .db file on your computer.

Python includes the sqlite3 module built-in — no installation needed.

import sqlite3

# 1. Connect (creates the file if it doesn't exist)
conn = sqlite3.connect("school.db")

# 2. Create a cursor — your helper for running SQL
cur = conn.cursor()

# 3. ... run SQL statements here ...

# 4. Save changes
conn.commit()

# 5. Close connection when done
conn.close()
🔑 The Flow: Connect → Cursor → Execute → Commit → Close. Commit saves your changes — forget it and nothing is saved!

📋 CREATE TABLE — Designing Storage

+

Before storing data, you need to create a table. Think of this as designing a spreadsheet — you decide the column names and what type of data each column holds.

cur.execute("""
    CREATE TABLE IF NOT EXISTS students (
        id      INTEGER PRIMARY KEY,
        name    TEXT NOT NULL,
        age     INTEGER,
        grade   REAL
    )
""")
conn.commit()

➕ INSERT — Adding Rows (CRUD: Create)

+ 
# Insert one row using ? placeholders (SAFE)
cur.execute(
    "INSERT INTO students (name, age, grade) VALUES (?, ?, ?)",
    ("Alice", 20, 85.5)
)
conn.commit()

# Insert multiple rows with executemany()
students = [
    ("Bob", 21, 78.0),
    ("Cara", 19, 92.3),
    ("Dan", 22, 65.0)
]
cur.executemany(
    "INSERT INTO students (name, age, grade) VALUES (?, ?, ?)",
    students
)
conn.commit()
⚠️ SECURITY — Use ? Placeholders! Never build SQL by gluing strings together:

❌ cur.execute(f"INSERT INTO users VALUES ('{name}')") — dangerous, allows SQL injection
✅ cur.execute("INSERT INTO users VALUES (?)", (name,)) — safe, escapes input properly
📝 Note: The ? placeholders expect a tuple, even for a single value. That's why you see (name,) — the trailing comma makes it a tuple with one item.

🔍 SELECT — Reading Data (CRUD: Read)

+ 
# Get ALL rows
cur.execute("SELECT * FROM students")
rows = cur.fetchall()
for row in rows:
    print(row)    # (1, 'Alice', 20, 85.5)

# Get ONE row
cur.execute("SELECT * FROM students WHERE id=?", (1,))
student = cur.fetchone()
print(student)

# Specific columns + WHERE condition
cur.execute(
    "SELECT name, grade FROM students WHERE grade > ?",
    (80,)
)
for name, grade in cur.fetchall():
    print(f"{name}: {grade}")

# ORDER BY — sort results
cur.execute("SELECT name, grade FROM students ORDER BY grade DESC")

✏️ UPDATE & DELETE (CRUD: Update & Delete)

+ 
# UPDATE — change existing rows
cur.execute(
    "UPDATE students SET grade=? WHERE name=?",
    (95.0, "Alice")
)
conn.commit()

# DELETE — remove rows
cur.execute(
    "DELETE FROM students WHERE age < ?",
    (18,)
)
conn.commit()

# How many rows were affected?
print(f"Rows deleted: {cur.rowcount}")
⚠️ DANGER: UPDATE or DELETE without a WHERE clause affects every single row in the table. Always double-check your WHERE condition before committing!

🔑 Remember: Every database program does some combination of CRUD. Master these four and you can build anything.

🛡️ Best Practices & Common Pitfalls

+
  • Always commit. Changes aren't saved to the file until you call conn.commit().
  • Always close. End with conn.close() to release the file and ensure all changes are flushed.
  • Always use placeholders. Never use f-strings or + to insert user input into SQL — use ? placeholders.
  • Handle errors. Wrap database code in try/except to catch sqlite3.IntegrityError (duplicate keys) and sqlite3.OperationalError (locked DB, bad SQL).
  • Use context managers for safety. with sqlite3.connect("db.db") as conn: automatically commits and closes.
# Full template — the safe way
import sqlite3

try:
    with sqlite3.connect("school.db") as conn:
        cur = conn.cursor()
        cur.execute("SELECT * FROM students")
        for row in cur.fetchall():
            print(row)
except sqlite3.Error as e:
    print(f"Database error: {e}")
