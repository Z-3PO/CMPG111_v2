Exam Guide

Strategy for tackling questions, the mistakes every first-year makes, and concept-check questions to test your understanding.

🧭 Part A: How to Approach Exam Questions

A step-by-step system that works for every question type

Step 1 · Read the ENTIRE Question First

Don't start coding after the first sentence. Read the full question — the last sentence often contains critical details (output format, edge cases, what the function must return). Underline key action words: "display", "return", "calculate", "store".

Step 2 · Identify the IPO

Before writing any code, figure out:

  1. Input — What data do you need? From the user? From a file? From a database?
  2. Processing — What must happen? A calculation? A decision? A loop? A database query?
  3. Output — What must be displayed/returned/saved?

Write these three words in the margin and jot down your answers. This takes 30 seconds and prevents 10 minutes of confusion.

Step 3 · Write Pseudocode Before Real Code

If the question is worth more than 5 marks, sketch the logic in plain English first:

GET price from user
GET quantity from user
CALCULATE subtotal = price × quantity
CALCULATE vat = subtotal × 0.15
DISPLAY subtotal, vat, and total

Once your pseudocode makes sense, translating to Python is mechanical. Most marks are for the logic, not the syntax.

Step 4 · Check Your Variable Types

Before any calculation, ask: "Is this variable a string or a number?" If it came from input(), it's a string. If you need maths, convert it. This single check prevents the #1 exam mistake.

Step 5 · Trace Through with Sample Values

Pick simple numbers (like 5 and 10) and mentally run through your code line by line. Write a trace table if the question involves a loop:

Iterationitotal
Start—0
1st11

3rd36

If the trace gives you the expected answer, your logic is correct.

Step 6 · Scan for Common Mistakes

Before submitting, quickly check:

  • Did I convert input() where needed?
  • Are my if/elif boundaries in the right order (highest first)?
  • Does my while loop have an update statement?
  • Did I use return (not print) when the question says "returns"?
  • Did I commit() after INSERT/UPDATE/DELETE?
  • Did I close my database connection?

🕐 Time Management Tips

  • Read all questions first. Answer the ones you're confident about, then tackle harder ones.
  • 1 mark ≈ 1-2 minutes. A 10-mark question shouldn't take more than 15-20 minutes.
  • Partial marks exist. Even if you can't solve the whole question, write what you can — the IPO, the function signature, the loop structure. Every correct line earns marks.
  • Don't debug one question for 30 minutes. If stuck, move on and come back later.

🚫 Part B: Common Mistakes

The errors first-years make every single year — and exactly how to fix them

📥 Input/Output Mistakes

+
❌ Forgetting to convert input()

input() ALWAYS returns a string. Writing age = input("Age: ") and then doing age + 5 gives a TypeError. Fix: age = int(input("Age: "))

❌ Using int() for decimal input

If the user might type 3.5, int(input()) crashes. Fix: Use float(input()) for any possibly-decimal number.

❌ Forgetting format specifiers

Question says "display to 2 decimal places" but you write print(total). Fix: print(f"R{total:.2f}").

❌ Missing the f in f-string

print("Hello {name}") literally prints {name}. Fix: print(f"Hello {name}") — the f before the quote is essential.

🔀 Decision Mistakes

+
❌ Using = instead of ==

if x = 5: is assignment (and a syntax error). Fix: if x == 5: (comparison).

❌ Wrong order of elif conditions

Checking mark >= 50 before mark >= 75 means 90% gets "Pass" not "Distinction". Fix: Always check from highest to lowest.

❌ Missing colons

Every if, elif, else, for, while, def line ends with :. Forgetting it is a SyntaxError.

❌ Using and when you mean or

"If the day is Saturday or Sunday" → if day == "Sat" or day == "Sun". Using and would require BOTH to be true (impossible!).

🔄 Loop Mistakes

+
❌ Infinite loops — forgetting the counter update

In a while loop, if you never change the variable in the condition, it stays True forever. Fix: Always include counter += 1 (or equivalent) inside the loop.

❌ Off-by-one errors in range()

range(1, 10) gives 1–9, not 1–10. Fix: range(1, 11) to include 10.

❌ Initialising accumulators inside the loop

Placing total = 0 inside the loop resets it every iteration. Fix: Initialise BEFORE the loop.

❌ Dividing by zero when computing averages

If a sentinel loop processes 0 items, total / count crashes. Fix: Check if count > 0 before dividing.

🧩 Function Mistakes

+
❌ Using print() when the question says "return"

If the question says "write a function that returns the total", use return total. Using print(total) means the caller gets None.

❌ Calling a function before defining it

Python reads top-to-bottom. The def must appear before the call.

❌ Forgetting to capture the return value

calculate_vat(200) computes the VAT but throws it away. Fix: result = calculate_vat(200).

📁 Files/Exceptions Mistakes

+
❌ Using "w" when you meant "a"

Mode "w" erases the file contents. If you want to add data to an existing file, use "a" (append).

❌ Forgetting \n in write()

Unlike print(), file.write() does NOT add a newline. Everything ends up on one line.

❌ Catching the wrong exception

If you use except ValueError but the actual error is FileNotFoundError, Python still crashes. Match exception types to the actual errors.

❌ Forgetting .strip() when reading lines

Lines from files include \n. Comparing "Alice\n" == "Alice" is False. Use line.strip().

📋 Lists/Tuples Mistakes

+
❌ Index out of range

A list of 5 items has indexes 0–4. Accessing list[5] crashes with IndexError. Remember: Last valid index = len(list) - 1.

❌ Trying to modify a tuple

Tuples are immutable. my_tuple[0] = 5 crashes. If you need to change data, use a list instead.

❌ Confusing sort() and sorted()

list.sort() sorts in-place and returns None. sorted(list) returns a new sorted list. Writing x = list.sort() gives x = None.

🗄️ Database Mistakes

+
❌ Forgetting conn.commit()

Without commit(), INSERT/UPDATE/DELETE changes are never saved. The data vanishes when you close the connection.

❌ Using f-strings in SQL (injection risk)

f"SELECT * FROM users WHERE name = '{name}'" is DANGEROUS. Use ? placeholders: "SELECT * FROM users WHERE name = ?", (name,).

❌ Forgetting the comma in single-value tuples

cur.execute("...WHERE id=?", (5)) is wrong — (5) is just the number 5 in brackets, not a tuple. Fix: (5,) — the trailing comma makes it a tuple.

❌ UPDATE/DELETE without WHERE

DELETE FROM students deletes ALL rows. Always specify WHERE to target specific records.

💡 Part C: Concept Check — Theory Questions

Not all exam questions require writing code. Test your understanding of the concepts behind the syntax.

Input/Output

C1 · Why does input() always return a string?

Explain in your own words why input() returns a string, and what a programmer must do if they need a number.

✅ Answer

Python can't know whether the user intended to type a number, a name, or an address — all keyboard input is just text (characters). That's why input() returns a string. The programmer must explicitly convert using int() or float() when they need to do maths. This is called type casting.

Decisions

C2 · What is the difference between = and ==?

✅ Answer

= is the assignment operator — it stores a value in a variable (x = 5 means "put 5 into x"). == is the comparison operator — it checks whether two values are equal and returns True or False (x == 5 means "is x equal to 5?"). Using = in an if statement causes a SyntaxError.

Loops

C3 · When would you use a while loop instead of a for loop?

✅ Answer

Use a for loop when you know in advance how many times to repeat (e.g., "process 10 students"). Use a while loop when you don't know — the loop depends on a condition (e.g., "keep asking until the user types 'quit'", or "keep dividing until the number reaches 1"). A for loop is count-controlled; a while loop is condition-controlled.

Functions

C4 · Explain the difference between a parameter and an argument.

✅ Answer

A parameter is the placeholder variable name in the function definition (e.g., def greet(name) — name is the parameter). An argument is the actual value you pass in when calling the function (e.g., greet("Alice") — "Alice" is the argument). Parameters are like empty boxes; arguments are what you put in them.

Functions

C5 · What happens if a function has no return statement?

✅ Answer

The function returns None by default. If you try to store or use the result (e.g., result = my_func()), result will be None. This is a common bug — if you need the function to produce a value for later use, you must use return.

Files/Exceptions

C6 · What does the finally block do and when would you use it?

✅ Answer

The finally block runs no matter what — whether the try block succeeded or an exception was caught. It's used for cleanup code that must always execute, such as closing files or database connections. Even if an error occurs, the finally block guarantees the cleanup happens.

Files/Exceptions

C7 · Why is the with statement preferred over manual open()/close()?

✅ Answer

The with statement automatically closes the file when the block ends — even if an exception occurs. With manual open()/close(), if an error happens between them, the file stays open and might get corrupted. with is safer, shorter, and considered best practice.

Lists/Tuples

C8 · What is the difference between a list and a tuple?

✅ Answer

Both store collections of items and support indexing/slicing. The key difference is mutability: lists ([]) can be changed after creation (add, remove, modify items), while tuples (()) are immutable — once created, their contents cannot change. Use lists for data that will be modified; use tuples for fixed data like coordinates, dates, or database rows.

Database

C9 · Why should you use ? placeholders instead of f-strings in SQL?

✅ Answer

To prevent SQL injection — a security attack where a user enters malicious SQL code as input. For example, typing ' OR 1=1 -- into a name field could trick the database into returning all records. The ? placeholder tells sqlite3 to safely escape the input, making injection impossible. This is called a parameterised query.

Database

C10 · What does conn.commit() do and when is it needed?

✅ Answer

commit() saves all pending changes to the database file on disk. Without it, any INSERT, UPDATE, or DELETE operations exist only in memory and are lost when the connection closes. It's needed after any operation that modifies data — but NOT after SELECT (which only reads). Think of it as pressing "Save" in a document editor.

Database

C11 · What is the difference between fetchone() and fetchall()?

✅ Answer

fetchone() returns a single row as a tuple (or None if no results). fetchall() returns ALL matching rows as a list of tuples. Use fetchone() when you expect exactly one result (e.g., looking up by primary key). Use fetchall() when you want multiple results to loop through.

Loops

C12 · What is a sentinel value and why is the input needed twice?

✅ Answer

A sentinel value is a special value that signals the end of input (like -1 or "quit"). The input appears twice because of the priming read pattern: you must read the first value BEFORE the loop starts (so the condition can be checked), and then read again at the END of each loop iteration to get the next value. Without the priming read, the loop either never starts or processes the sentinel as real data.
